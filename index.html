lv_circuitpython_mod

CircuitPython integration for LVGL: tree patches, build glue (build_cp.sh), spike templates, and tests. It consumes lv_bindings CircuitPython artifacts (generated/lvgl_circuitpython.c / .h) so import lvgl works as a CircuitPython-native module.

This is not a MicroPython USER_C_MODULES usermod — CircuitPython uses a different module layout and patch-driven build. Pair a stable CircuitPython release tag with regenerated bindings, then drive any port/board/variant through ./build_cp.sh.

First-time setup

Sibling clones of lv_bindings/ and a tagged circuitpython/ 
# Pick a stable tag from github.com/adafruit/circuitpython/releases
git clone --branch 10.2.1 \
  https://github.com/adafruit/circuitpython.git circuitpython
cd circuitpython && make fetch-all-submodules && cd ..

git clone https://github.com/PyDevices/lv_bindings.git lv_bindings
cd lv_bindings
git submodule update --init lvgl
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
./regenerate_lvcp.sh
cd ..

Patch and build

build_cp.sh always runs apply_cp_lvgl_patches.sh --apply first

unix coverage

Default cmods cp-unix target; good for the CP test suite / gcov.

cd lv_circuitpython_mod
./apply_cp_lvgl_patches.sh --apply \
  --port unix --variant coverage
./build_cp.sh --port unix --variant coverage

../circuitpython/ports/unix/build-coverage/micropython \
  ./test_lvgl_cp_unix.py

Board firmware

Interactive or CLI port/board/variant selection. Install GCC 14+ toolchains first (see README).

./build_cp.sh --port espressif \
  --board espressif_esp32p4_function_ev

./build_cp.sh   # interactive selection
